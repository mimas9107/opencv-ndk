# OCR ROI Gate 調整紀錄

## 調整前

- 最小 ROI 門檻: `32x32`
- 上限 ROI 門檻: `256x256`

## 實機現象

- 主文字 `logitech` 可穩定辨識
- 同一場景偶爾會出現次要候選框，辨識成單字元 `t`
- 該次要候選框尺寸約為 `60x66` 或 `59x62`

## 調整後

- 最小 ROI 門檻: `64x64`
- 上限 ROI 門檻: `256x256`

## 原因

- 第一版先收斂 ROI gate，避免過早更換模型或字庫
- `logitech` 主框約 `190x68`，仍可通過新門檻
- 次要小框會更容易被 gate 擋下，降低 UI 顯示噪音
